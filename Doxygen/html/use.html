Use

Implementations of the hardware abstractions (like pins and delays) are provided for the supported targets:

  • Arduino Uno (ATMEGA328P chip)
  • Arduino Due (ATSAM3X8E chip)
  • DB103 (LPC1114 chip)
  • blue-pill (stm32f103c8 chip)

The easy way is to use the library with bmptk: Include hwlib.hpp, which will include the correct target header based on the HWLIB_TARGET_* macro that is set in the bmptk makefile. Bmptk passes this setting to the hwlib.hpp file via a macro definition on the compiler command line.

A hwlib application includes hwlib.hpp.

This file will include the correct target header based on the HWLIB_TARGET_* macro., defined in the compiler command line. This macro must identify one of the supported targets .

When the bmptk build system is used, the HWLIB_TARGET_* passed is determined by the TARGET setting in the bmptk makefile.

The sources within the library use a makefile.link file in each subdirectory, which passes 'control' up to the next-higher directory.

The makefile.link in the root directory includes the makefile.custom (if one exsts) or the makefile.local, which set things like the location of the bmptk and catch2 files.
