Detailed Description

This file contains two data objects that handle and trim the outgoing and received commands of a pn532.

This file provides data objects that help standardize the received pn532 commands to a fixed format. Normally, the pn532 sends a frame to the host controller with information in response to a command send. However, depending on what interface is used, the frame might look a little different.

A general pn532 frame looks like:

0x00, 0x00, 0xFF, LEN, LCS, TFI, PD0, PD1, PDn, DCS, 0x00

Where:

  • 0x00, 0x00, 0xFF = preamble (start of a command)
  • LEN = Length of the receiven command
  • LCS = Checksum of lenght
  • TFI = Frame identifier ( D4: host -> pn532. D5: pn542 -> host)
  • PD0, PD1, PDn = Packed data
  • DCS = Packed data Checksum
  • 0x00 = Postamble (end of a command)

For more detailled information, please refer to the user manual: https://www.nxp.com/docs/en/user-guide/141520.pdf P. 28 - 6.2.1.1
